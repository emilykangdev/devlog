# Learnings — March 13 2026

## Technical

- Pane + Claude Code Max is a legitimately better setup than stitching together Augment/ChatGPT/Codex manually
- `/discussion` → `/plan` → `/implement` → `/prepare-pr` is the workflow — quality of planning determines quality of code
- MCP servers added: Context7, Stripe, Linear, Supabase
- Custom slash commands live in `.claude/commands/`, skills in `.claude/skills/`
- `CLAUDE.md` is persistent memory — use it to enforce Context7 for all API lookups
- Deepgram auth fix: use `createClient(key)` not `DefaultDeepgramClient({ apiKey })`
- 422 is the right status code for a valid URL that isn't a valid RSS feed
- `DefaultDeepgramClient` is old v2 pattern, `createClient` is correct v3

## Product

- v1 scope: RSS URL → transcript → Claude brief → email via Resend. That's it
- Google OAuth + Supabase + Stripe $9/mo + 3 day trial is the auth/payments stack
- Drop speaker attribution entirely — legal risk not worth it for v1
- Drop Browserbase — Exa search API is simpler, faster, good enough for link finding
- Plans that take 2+ hours to review are too big — split them

## Mindset

- "I keep facing this damn problem" is your entire pitch
- Do it yourself when AI has cycled on the same fix 2-3 times
- No tests until you have paying users
- Ship first, iterate on depth vs brevity once real users tell you what's missing
- The plan is the blueprint — bad plan means bad code, every time. YOU NEED TO UNDERSTAND EVERY SINGLE FEATURE AND HOW THEY CONNECT TO EACH OTHER. Not about syntax, but generally you should be able to understand the gist of your own codebase, and make sure it's clean. Can't have any uncertainties and leave it up to AI to decide, because it'll get it wrong every single time.
